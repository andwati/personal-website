# Getting Started with Nuxt 4

Nuxt 4 represents a significant evolution in the Nuxt ecosystem. With a refined project structure, improved developer experience, and powerful new features, it's never been a better time to start building with Nuxt.

## What's New in Nuxt 4

Nuxt 4 brings several exciting improvements:

- **New directory structure** — The `app/` directory is now the default home for your Vue application code
- **Improved TypeScript support** — Better type inference and auto-completion out of the box
- **Enhanced performance** — Faster builds and smaller bundle sizes
- **Better data fetching** — Reworked async data layer for more predictable behavior

## Getting Started

Setting up a new Nuxt 4 project is straightforward:

```bash
npx nuxi@latest init my-app
```

This creates a fresh project with all the latest defaults. From there, you can start building your application right away.

## Why Nuxt?

Nuxt takes the best of Vue.js and adds powerful conventions and features on top:

1. **File-based routing** — No manual route configuration needed
2. **Auto-imports** — Components, composables, and utilities are automatically available
3. **Server-side rendering** — Built-in SSR for better performance and SEO
4. **Modules ecosystem** — Rich ecosystem of modules for common needs

Whether you're building a simple landing page or a complex full-stack application, Nuxt provides the tools and conventions to do it efficiently.

## Conclusion

Nuxt 4 is a fantastic framework for modern web development. Give it a try and see how it can streamline your workflow!
